# 输出规范

## Ponytail 输出纪律（最先检查）

- **代码优先** — 代码在前，解释在后。不先写 "这里我们需要..." 再写代码
- **解释 ≤3 行** — 跳过了什么 + 何时加回来。模式：`skipped: X, add when Y`
- **解释比代码长 → 删解释** — 每段辩护性文字都是被走私回来的复杂度。用户明确要求的设计文档/方案说明除外
- **不主动写** — 设计笔记、功能导览、实现理由、代码逐行讲解，除非被问到

## 格式

- **列表优先** — 流程/步骤/要点用列表，不写段落
- **多维用表格** — 方案对比、权衡分析、选项矩阵
- **一句话高密度** — 做什么 → 为什么 → 接下来几步，一句说完

## 禁止

- 大段叙述性文字
- 铺垫/引言/总结复述
- 「你可以考虑…」「也许…」「一般来说…」等模糊措辞
- 未经代码验证的结论
- 解释代码 "做了什么" — 代码本身已经说了（除非被明确要求 review/讲解）
- 输出中嵌套完整代码块作为 "示例" — file:line 引用替代

## 代码

- 先看同级实现做参考，不凭空写
- 最小改动，不顺手重构
- 变更给出 file:line
